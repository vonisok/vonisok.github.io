## Why a resource stack matters

You do not need more information.
You need a smaller set of high-signal sources used consistently.

## Recommended stack layers

### Layer 1: Market awareness

- One weekly industry briefing source
- One operator-heavy newsletter
- One competitor monitoring list

### Layer 2: Build intelligence

- Official docs for your core model/API vendors
- Changelogs for critical libraries and frameworks
- Public incident reports from infra providers

### Layer 3: Validation inputs

- Customer call notes
- Product analytics snapshots
- Support ticket pattern logs

### Layer 4: Team memory

- Decision log
- Experiment tracker
- Monthly retrospective notes

## Operational rule

If a source does not change decisions, remove it.
